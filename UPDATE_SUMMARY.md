## 主要更新

### 新增功能：用户配置文件清理选项

现在脚本会询问用户是否清理以下文件：

1. **`.claude.json`** - 用户配置文件
   - 包含内容：User ID、使用统计、提示历史、主题设置等
   - 删除影响：使用统计重置，提示历史清空
   - 适用场景：想要重置使用记录或切换用户

2. **`.credentials.json`** - 认证凭据
   - 包含内容：登录凭据
   - 删除影响：需要重新登录 Claude Code
   - 适用场景：切换账号或清理登录信息

### 交互界面改进

**新的选项界面**：
```
是否清理用户配置和认证文件？

  [1] .claude.json - 用户配置（包含 User ID、使用统计等）
      删除后会重置使用统计和提示历史

  [2] .credentials.json - 认证凭据
      删除后需要重新登录 Claude Code

  [3] 两者都清理
  [4] 都不清理（推荐）

请输入选项 (1-4):
```

### 安全机制

- ✅ 默认选项为"都不清理"（推荐）
- ✅ 清晰说明每个文件的作用和删除后果
- ✅ 在干运行模式下会显示预览信息
- ✅ 执行模式下会显示删除确认信息

## 注意事项

1. **User ID 说明**
   - `.claude.json` 中的 User ID 是匿名标识符
   - 用于统计和遥测，不包含个人信息
   - 删除后会生成新的 User ID

2. **使用统计**
   - 删除 `.claude.json` 会清空所有使用统计
   - 包括：消息数、会话数、Token 使用量等
   - 如需保留统计，请选择"都不清理"

3. **认证凭据**
   - 删除 `.credentials.json` 后需要重新登录
   - 如果使用自定义 API 端点，需要重新配置
   - 建议在确认需要切换账号时再清理
